## כשמגיעים קבצים חדשים — רשימת בדיקות

### ✅ דברים שעובדים אוטומטית (לא צריך לשנות)

| מה | איך |
|----|-----|
| **שנה שונה** (2027, 2028...) | `infer_year_from_workbook()` מוצא את השנה הנפוצה בקובץ |
| **מתרגלים חדשים** | נוספים אוטומטית לדוח; מסווגים לפי `DEFAULT_SUBJECT` אם לא ברשימה |
| **שמות תלמידים עם שגיאות כתיב** | `names_match()` עם `SequenceMatcher` + סף נמוך לאותו שם פרטי |
| **פורמטים שונים של זמן** | `is_time_marker()` תומך ב-`dt_time`, `HH:MM`, `HH:MM:SS`, `HH:MM - HH:MM` |
| **גלישת טקסט** | `note_row_height()` + `tutors_row_height()` מחשבים אוטומטית |
| **מיקומים עם הערות** | `find_known_location()` מחלץ רק מה שב-`KNOWN_LOCATIONS` |

## אלגוריתמים מרכזיים

### חילוץ מיקום (`extract_location`)

```
שם מתרגל גולמי
  ↓ הסר "מחליפ(ה) את X"
  ↓ הסר סוגריים
  ↓ פצל לפי \n → קח סגמנט ראשון → פצל לפי מקף
  ↓ עבור כל חלק: find_known_location()  ← מחפש רק מה שב-KNOWN_LOCATIONS
  ↓ אם לא נמצא: בדוק סגמנטים הבאים (שורות חדשות)
  ↓ אם לא נמצא: בדוק מילים אחרי השם הראשון
  → None אם לא נמצא מיקום ידוע
```

`find_known_location` תמיד בודק **רב-מילי לפני חד-מילי** (למנוע "צור" → צורן במקום צור יצחק).

### fuzzy match (`names_match`)

⚠️ **לעולם לא** להשתמש ב-`sum(c in b for c in a)` — אותיות עבריות נפוצות (י,ל,ו,ר) יוצרות false positives.

```python
# אותו שם פרטי → סף 0.70 (מכסה שגיאות כתיב בשם משפחה)
# שמות שונים  → סף 0.85
difflib.SequenceMatcher(None, an, bn).ratio()
```

### זיהוי שנה (`infer_year_from_workbook`)
```python
# סורק 3 שורות ראשונות של כל גיליון, מוצא את השנה הנפוצה ביותר
# לא מניח 2026 — עובד עם כל שנה
```

### זיהוי מקצוע לפי תווית סלוט
```
"14:30 אנגלית" → current_subject = 'אנגלית'
"16:00 מתמטיקה" → current_subject = 'מתמטיקה'
"19:00:00" (time object) → נשאר כמקצוע הקודם (subject propagation)
```

## לוגיקת ההשוואה (`run_comparison`)

ההשוואה עובדת בשלושה שלבים:

### שלב 1 — סשנים משותפים (יש ב-ייצוא וגם בדרוש)

לכל סשן ב-ייצוא שיטס `(תאריך, קנון, שעה)`:
- מחפש מפתח תואם בדרוש תיקון. שני מנגנוני fallback אם key לא נמצא:
  1. **fuzzy שם מתרגל**: אותו תאריך + אותה שעה, partial name match
  2. **fuzzy שעה** (±15 דק'): אותו תאריך + אותו קנון בדיוק, שעה קרובה ≤15 דקות
- משווה רשימות תלמידים:
  - תלמיד בייצוא אך לא בדרוש → **להוסיף**
  - תלמיד בדרוש אך לא בייצוא → **להסיר**
- המפתחות שנבדקו נשמרים ב-`processed_darush_keys`

### שלב 1.5 — עמודות ריקות בייצוא שיטס → `empty_sessions`

במהלך פרסור ייצוא שיטס: אם עמודה לא הניבה אף תלמיד (`col_had_students=False`), ועבור אותו `(תאריך, קנון)` יש בדרוש סשנים — מוסיפים ל-`empty_sessions`:
```python
{'date', 'canon', 'tutor', 'subject', 'location', 'count'}
# count = מספר הסשנים בדרוש תיקון עבור אותו (תאריך, מתרגל)
```
**בדוח**: שורות צהובות בולטות עם הטקסט "ייצוא שיטס ריק — יש X שיעורים בדרוש תיקון. יש למלא בייצוא."

### שלב 2 — סשנים שקיימים רק בדרוש תיקון

אחרי לולאת הייצוא, עוברים על כל `(תאריך, קנון, שעה)` בדרוש שלא ב-`processed_darush_keys`:
- **מדלגים** אם `(תאריך, קנון)` ב-`empty_covered` (כבר מדווח כ-empty_session)
- **מדלגים** אם התאריך לא ב-`yitzua_sheet_dates` (גיליון לאותו שבוע לא קיים בייצוא — שבוע עתידי)
- אחרת: כל תלמיד בסשן → **להסיר**

**דוגמאות:**
| מצב | תוצאה |
|-----|--------|
| עמודה ריקה למתרגל בייצוא (נגה 01.06) | שורת ⚠️ צהובה — "ייצוא שיטס ריק — יש 3 שיעורים..." |
| שעות חופפות בייצוא 2 מתוך 4 (יניב 01.06) | 2 שעות שרק בדרוש → להסיר |
| ייצוא=16:30, דרוש=16:40 — mismatch קטן | fuzzy time match → משווה נכון |
| גיליון ייצוא לשבוע 21-30.06 לא קיים | מדלג לחלוטין — לא מדווח |

## מקרי קצה ידועים

| מקרה | טיפול |
|------|-------|
| `19:00:00` כ-`dt_time` object | `isinstance(val, dt_time)` בתחילת `is_time_marker` |
| `"ירקון התבטל מחלה"` | `find_known_location` מחלץ `ירקון` ומתעלם מהשאר |
| `"זום-סמרה"` | `סמרה` לא ב-`KNOWN_LOCATIONS` → מחזיר `None` (נכון, סמרה = שם) |
| `"אדיר\nתל מונד +\n ניצני עוז"` | מחזיר `תל מונד` (ראשון) ולא `ניצני עוז` |
| `=counta(...)` | `is_junk()` מסנן נוסחות Excel |
| שמות עם מקף כמו `"ארבל-לוי"` | `clean()` מסיר מקף לפני השוואה |
| כיתה בסוף שם `"יהלי לוי - ח"` | `clean()` מסיר `[חטיכזאבדה]\d*$` |

## גובה שורת כותרת — חישוב דינמי ללא גלישה

שורה 2 מכילה שני תאים שחייבים להתאים לאותו גובה שורה ולא לגלוש:
- **B2**: שם המקצוע ("מתמטיקה" / "אנגלית") — col צר (14 units)
- **D2:G2**: שמות המדריכים — col רחב (96 units)

```python
# כל תא מקבל גודל פונט משלו — B2 חייב להיות בשורה אחת, D2:G2 יכול ל-3 שורות
def _fit_font(text, col_units, max_lines, char_ratio=0.7, px_per_unit=7.2):
    col_px = col_units * px_per_unit
    for size in range(22, 7, -1):
        chars_per_line = max(1, int(col_px / (size * char_ratio)))
        if math.ceil(len(text) / chars_per_line) <= max_lines:
            return size
    return 8

def _cell_height(text, col_units, font_size, char_ratio=0.7, px_per_unit=7.2):
    col_px = col_units * px_per_unit
    chars_per_line = max(1, int(col_px / (font_size * char_ratio)))
    return int(math.ceil(len(text) / chars_per_line) * font_size * 1.6 + 10)

def calc_header_font_size(subject_text, tutors_text):
    """מחזירה (גודל_B2, גודל_D2G2, גובה_שורה).
    - B2 (col 14 units): חייב להיכנס ל-1 שורה בלבד
    - D2:G2 (col 96 units): יכול להתפרס על 3 שורות"""
    sz_b  = _fit_font(subject_text, 14, max_lines=1)
    sz_dg = _fit_font(tutors_text,  96, max_lines=3)
    row_h = max(_cell_height(subject_text, 14, sz_b),
                _cell_height(tutors_text,  96, sz_dg)) + 8
    return sz_b, sz_dg, int(row_h)

# שימוש:
sz_b, sz_dg, row2_h = calc_header_font_size(title, tutors_text)
ws.row_dimensions[2].height = row2_h
# B2 ו-D2:G2 מקבלים גדלים שונים — wrap_text=True חובה על שניהם!
# char_ratio=0.7 (Calibri) — מבטיח שלא תהיה גלישה גם בעמודות צרות
```

**כלל:** גובה השורה = `max(גובה_B2, גובה_D2G2) + 10pt padding`

## הרחבת צבעים לסניפים חדשים

כשמגיע סניף חדש שאינו ב-`KNOWN_LOCATIONS`, הדוח יציג `default` (אפור בהיר).
כדי להוסיף סניף עם צבע מותאם:

```python
# 1. הוסף לרשימה הידועה
KNOWN_LOCATIONS.add('באר שבע')

# 2. הגדר צבע (hex RGB ללא FF prefix)
BRANCH_COLORS['באר שבע'] = 'A8C4E0'   # כחול בהיר

# 3. הוסף לסרגל הצד (אם רוצים שיופיע בעמודה B)
BRANCHES_ORDER.append('באר שבע')
```

**כינוי לסניף קיים** (למשל "בי"ס הרצוג" = "אלתרמן"):
```python
LOCATION_NORMALIZE['הרצוג'] = 'אלתרמן'
# לא צריך צבע חדש — ישתמש בצבע של אלתרמן
```

**לבחור צבע:** השתמש בגוון דומה לסניפים הקיימים (אפורים-ירוקים-כחולים עמומים) כדי לשמור על עיצוב אחיד.

## סטטוס נוכחות תלמידים

### הגדרות
| צבע ב-ייצוא שיטס | משמעות | בדוח |
|---|---|---|
| 🟢 ירוק (FF00FF00, FFB6D7A8, FFD9EAD3...) | הגיע לשיעור | `שם 🟢` |
| 🔴 אדום (FFFF0000, FFEA9999...) | לא הגיע ולא הודיע | `שם 🔴` |
| ⬜ ללא צבע / לבן / צהוב | המתרגל לא סימן | `שם ⬜יש לסמן` |

### קוד

```python
GREEN_RGBS = {'FF00FF00','FFB6D7A8','FFD9EAD3','FF9AC47A','FF64FF00'}
RED_RGBS   = {'FFFF0000','FFEE0000','FFEA9999','FFF4CCCC'}

def get_attendance(rgb):
    r = str(rgb).upper() if rgb else '00000000'
    if r in GREEN_RGBS: return 'הגיע'
    if r in RED_RGBS:   return 'לא הגיע'
    return 'לא סומן'
```

### איפה הצבע נקרא
`parse_slots()` קורא את צבע כל תא תלמיד ושומר `{name: attendance}` במקום `{name}`.
הסטטוס מועבר לכל diff תחת מפתח `'attendance'`.

### בדוח
```python
ATTENDANCE_ICON = {'הגיע': '🟢', 'לא הגיע': '🔴', 'לא סומן': '⬜'}

def format_student(d):
    icon = ATTENDANCE_ICON.get(d.get('attendance'), '')
    if d.get('attendance') == 'לא סומן':
        return f"{d['student']} {icon}יש לסמן"
    return f"{d['student']} {icon}"
```

**להסיר** — לא מציגים סטטוס (התלמיד לא ב-ייצוא שיטס).
**להוסיף** — מציגים סטטוס מייצוא שיטס.

## ניקוי שמות תלמידים (`clean_student_name`)

שמות תלמידים מגיעים לעיתים עם תוספות שאינן חלק מהשם האמיתי:

| דוגמה גולמית | שם נקי | סוג רעש |
|---|---|---|
| `עומר פילוס עד 10.5 (לבנתיים)` | `עומר פילוס` | הערה + תאריך |
| `מורן אשר 21.1 ח הדר עם` | `מורן אשר` | תאריך + כיתה + מיקום |
| `שליו פרידלנדר-ח` | `שליו פרידלנדר` | כיתה מחוברת |
| `הדר ילוז - י - 3.12 - בית יצחק` | `הדר ילוז` | כיתה + תאריך + מיקום |
| `שחר דוידוב?` | `שחר דוידוב` | סימן שאלה |
| `יעל קורדובר (לא נכנסת למערכת)` | `יעל קורדובר` | הערה בסוגריים |
| `רומיה אקב - ויתרתי לה על חיסור` | `רומיה אקב` | הסבר ארוך |

### האלגוריתם

```python
HEBREW_GRADES = {'א','ב','ג','ד','ה','ו','ז','ח','ט','י','יא','יב','יג'}
NOISE_WORDS = {
    'שדי','חמד','ויתרתי','עד','בגלל','בדיקה','רפואית','לה','על','חיסור',
    'בית','יצחק','שרונה','גאולים','ירקון','צורן','זום','רופין','חגור','מתן',
    'תל','מונד','אלתרמן','ניצני','עוז','עמי','אסף',
}

def clean_student_name(raw):
    s = str(raw).strip().replace('?', '')
    s = re.sub(r'\([^)]*\)', '', s).strip()          # הסר (הערות)
    s = re.sub(r'[״׳]', '', s).strip()               # הסר גרש
    s = re.sub(r'-([א-ת]{1,2})(?=\s|$)', r' \1', s) # "פרידלנדר-ח" → "פרידלנדר ח"
    name_words = []
    for word in s.split():
        w = word.strip("'-,.")
        if w in HEBREW_GRADES: break      # אות כיתה → עצור
        if re.search(r'\d', w): break     # ספרה → תאריך/כיתה → עצור
        if w in NOISE_WORDS: break        # מילת רעש → עצור
        if len(w) > 12 and len(name_words) >= 2: break
        name_words.append(word)
    return re.sub(r'\s+', ' ', ' '.join(name_words).strip(" '-,."))
```

**⚠️ חשוב:** `clean_student_name` מופעל על **שני הקבצים** בזמן פרסור, לפני ההשוואה. כך `עומר פילוס עד 10.5` ו-`עומר פילוס` הופכים לאותו מחרוזת ומתאימים נכון.

### הרחבה עתידית
```python
# מילת רעש חדשה:
NOISE_WORDS.add('המילה')

# כיתה חדשה (מעל יג):
HEBREW_GRADES.add('יד')
```

## כללי סינון מיוחדים בייצוא שיטס

### 1. "(לא נכנס/ה למערכת)" — דלג על התלמיד
תלמיד עם הערה זו רשום בידיעת המתרגל אך **אינו במערכת** — לכן אין טעם לדווח עליו כפער שצריך להוסיף.

```python
# בתוך parse_slots:
if 'לא נכנס' in vs:
    continue   # דלג, לא להוסיף לסלוט
```

### 2. שמות עם **קו חוצה (strikethrough)** — דלג
כשתלמיד מסומן בקו חוצה בייצוא שיטס, המשמעות: **הודיע מראש על היעדרות**. לא נחשב פער.

```python
if cell.font and getattr(cell.font, 'strike', False):
    continue
```

### 3. מתרגלים שמלמדים מקצוע אחד בלבד
חשוב לשמור את `MATH_TUTORS` ו-`ENG_TUTORS` מעודכנים — מתרגל ברשימה הלא נכונה יגרום ל-false positives.

**נכון:** `נועה` ו-`נעה` שייכות לאנגלית בלבד (לא ב-MATH_TUTORS).

## זיהוי שעות (גנרי)

`is_time_marker` מזהה תא כשורת זמן אם:
1. הוא Python `dt_time` object, או
2. הוא בפורמט שעה טהורה (`14:30`, `14:30:00`, `14:30-16:00`), או
3. **הוא מכיל `HH:MM` והוא קצר** (≤25 תווים) — לא משנה אילו מילים אחרות מופיעות

```python
def is_time_marker(val):
    if isinstance(val, dt_time): return True
    v = str(val).strip()
    if re.match(r'^\d{1,2}:\d{2}(:\d{2})?(\s*[-–]\s*\d{1,2}:\d{2})?$', v): return True
    if re.search(r'\d{1,2}:\d{2}', v) and len(v) <= 25:
        return True
    return False
```

**הרעיון:** שמות תלמידים אמיתיים לעולם לא מכילים `HH:MM`. לכן כל תא שמכיל שעה וקצר מספיק — חייב להיות שורת זמן, גם אם יש לידה הערה לא ידועה כמו `13:30 - שיעור השלמה` או `15:00 קבוצה ב'`.

**יתרון:** הסקיל **לא תלוי** ברשימה קבועה של מקצועות/מיקומים לזיהוי שעות. כל פורמט חדש שיופיע בעתיד יזוהה אוטומטית.

## כיול גודל הפונט — מניעת גלישה

### הבעיה
Excel רנדור פונטים בצורה לא מדויקת — `char_ratio` נמוך מדי גורם לסקיל לחשוב שטקסט נכנס לתא בשורה אחת בזמן שבפועל הוא **נחתך לשתי שורות** (למשל "מתמטי / קה").

### הפתרון
`char_ratio = 0.9` (שמרני). המשמעות: הסקיל מניח שכל תו עברי תופס 90% מגודל הפונט בפיקסלים.

**דוגמה:** עמודה B ברוחב 14 units ≈ 101px:
- `מתמטיקה` (7 תווים) → גודל 16pt (תוֹו ≈ 14.4px × 7 = 100px ✓)
- `אנגלית` (6 תווים) → גודל 18pt (תוֹו ≈ 16.2px × 6 = 97px ✓)

### למה לא להגדיל את עמודה B?
זה היה אומר שינוי כל הפריסה של הדוח. עדיף להתאים את הפונט.

### מה אם המקצוע יהיה ארוך יותר בעתיד?
למשל "ספרות והבעה" (10 תווים) — הסקיל יוריד אוטומטית את הפונט עד 11-12pt. אם זה יהיה קטן מדי, אפשר להגדיל את `col_widths[1]` מ-14 ל-20:

```python
col_widths = [3, 20, 3, 16, 56, 10, 14]  # עמודה B הוגדלה
```

## פונט הדוח ומניעת גלישה — Calibri + shrink_to_fit

### בחירת פונט
הפונט הראשי הוא **Calibri** (פונט סטנדרטי, נקי וקריא בעברית ובלועזית). ניתן לשנות:
```python
MAIN_FONT = 'Calibri'   # או 'Arial', 'Heebo', וכו'
```

### למה shrink_to_fit?
חישוב מתמטי של גודל פונט שלא יגלוש הוא לא מדויק (תלוי בגרסת Excel, מערכת הפעלה, וכיול הפונט הספציפי). לכן **תא B2 משתמש ב-`shrink_to_fit=True`** של Excel — תכונה שאומרת ל-Excel עצמו לכווץ את הפונט אוטומטית עד שהטקסט נכנס לתא. **זו הדרך היחידה להבטיח 100% שאין גלישה.**

```python
c.alignment = Alignment(horizontal='center', vertical='center',
                        wrap_text=False, shrink_to_fit=True, readingOrder=2)
```

**הגיון:**
- `_fit_font` מחשב גודל התחלתי סביר (למשל 22pt)
- Excel מכווץ אותו אוטומטית אם צריך (למשל ל-19pt עבור "מתמטיקה")
- אין `wrap_text` ב-B2 → אין חיתוך באמצע מילה

### מה לגבי D2:G2 (שמות המדריכים)?
שם זה לא רלוונטי — הטקסט ארוך מדי וצריך לעטוף לכמה שורות. לכן D2:G2 משתמש ב-`wrap_text=True` עם גובה שורה מחושב דינמית.

### לסיכום — אסטרטגיה כפולה:
| תא | טקסט | מנגנון |
|---|---|---|
| B2 | קצר ("מתמטיקה") | `shrink_to_fit=True` — Excel מכווץ אוטומטית |
| D2:G2 | ארוך (רשימת מדריכים) | `wrap_text=True` + גובה דינמי |
| הערות בשורות התלמידים | בינוני | `wrap_text=True` + `note_row_height()` |
